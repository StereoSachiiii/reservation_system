# 🔍 Frontend Audit Report: Vendor Application

This report outlines the technical and functional issues discovered during the audit of the `frontend/src/apps/vendor` module.

## 🔴 Critical Issues

### 1. Orphaned Checkout Route
The `CheckoutPage.tsx` exists in the file system but is **missing from `VendorRoutes.tsx`**.
- **Impact**: Vendors cannot complete payments for their reservations through the standard application flow.
- **Recommendation**: Add `/checkout/:id` to `VendorRoutes.tsx`.

### 2. Broken Navigation Links
Several pages reference routes that do not appear to be the correct landing pages for the vendor app.
- **`/home`**: Referenced in `VendorReservationDetailPage.tsx` and `CheckoutPage.tsx` as a fallback.
- **`/events`**: Referenced in `VendorDashboardPage.tsx` for the "Book Now" action.
- **Impact**: Users encounter 404s or get lost when errors occur or when trying to book.

## 🟡 Architectural Issues

### 3. State Management Inconsistency
There is a split in how data is fetched and managed:
- **`react-query`**: Used in `VendorReservationDetailPage.tsx` and `CheckoutPage.tsx`.
- **Manual `useEffect` + `useState`**: Used in `VendorDashboardPage.tsx`, `DocumentsPage.tsx`, and `VendorProfilePage.tsx`.
- **Impact**: Makes the codebase harder to maintain and results in inconsistent loading/error behaviors across the app.

### 4. Navigation Method Inconsistency
- `VendorProfilePage.tsx` uses `window.location.href = '/vendor/dashboard'`.
- Other pages use `useNavigate()` from `react-router-dom`.
- **Impact**: `window.location.href` causes a full page reload, breaking the Single Page Application (SPA) experience and losing state.

## ⚪ Technical Debt & UX

### 5. Hardcoded Constants
The `AVAILABLE_CATEGORIES` list is hardcoded inside `VendorReservationDetailPage.tsx`.
- **Impact**: If categories change in the backend, they must be manually updated in the frontend code.

### 6. Basic Loading States
`DocumentsPage.tsx` uses a simple `<div>Loading...</div>` while other pages use a high-fidelity animated spinner.
- **Impact**: Inconsistent "premium" feel across the application.

### 7. Profile Category Logic
`VendorProfilePage.tsx` explicitly removes category editing but preserves them in the save payload:
```typescript
const payload = { ...formData, categories: user?.categories || [] };
```
- **Impact**: Vendors cannot update their categories in their main profile page, only through a hidden modal in the reservation detail page.
